# Sksir New 优化计划清单

更新日期：2026-07-15

## 核心方向

这是一个静态个人导航页，不按复杂 Web 应用或安全服务来规划。后续只围绕三个目标做取舍：

- 首次启动快：PC 和手机都尽量在 0.5 秒内完成主体结构，缓存后必须稳定低于 0.5 秒。
- 动画好看：启动、分类、卡片、设置面板的动效要轻、顺、干净，高配保留完整观感，低配自动降级。
- 体验良好：移动端不抖、不缩放、不露底；收藏分类清楚；日常维护简单。

## 当前口径

- 站点：`https://sksir.top/`
- 项目用途：自用网页收藏、导航、搜索入口。
- 日常检查命令：`node scripts\check.js`
- 版本号：本计划不自动改版本号，需要发布时再运行 `node scripts\update-version.js YYYY.MM.DD.N`。
- 首屏口径：时间、搜索框、主体布局先完成；启动遮罩短暂等待壁纸结果；远程图标、访问统计、状态检测、更新检查不进入首屏等待。

## 下一轮优先级

1. iOS Safari 壁纸铺满
   - 目标：不露灰底，不影响 PC，不改变桌面端壁纸裁切。
   - 已在 `css/mobile.css` 的 `html.ios-safari` 作用域和早期高度变量内处理。
   - 每次部署后用 iPhone 15 Pro Max Safari 实测，注意 Service Worker 缓存是否已更新。

2. 首次启动体验
   - 目标：缓存后稳定低于 500ms，冷启动尽量接近 500ms。
   - 保持首屏 JS/CSS 轻量；隐藏书签分组、远程图标、统计、状态检测、更新检查继续后置。
   - 如果冷启动慢，优先检查壁纸、字体、图标和首屏 CSS，而不是引入新框架。

3. 启动动画和过渡
   - 目标：用户看到的是加载动画到完整页面的自然显现，不看到页面拼装过程。
   - 已完成轻量光环和主体显现过渡，继续避免大图、重滤镜、复杂 canvas。
   - 高配可保留更完整的过渡，低配和减少动态效果用户走轻量模式。

4. 移动端体验
   - 目标：搜索框不自动放大，键盘收起不缩放，分类栏不产生多余空白。
   - 搜索建议已加短防抖，减少移动端连续输入时的请求和面板抖动。
   - 移动端修复优先写在 `css/mobile.css` 或 `html.ios-safari` 作用域下。
   - 复测范围：iPhone Safari、普通安卓浏览器、PC 浏览器。

5. 收藏维护体验
   - 目标：分类清楚、标题短、描述短，日常新增站点不容易出错。
   - 继续用 `data/sites.js` 集中维护站点，改完运行 `node scripts\check.js`。
   - 后续如果收藏变多，再考虑把搜索引擎、壁纸配置拆成独立数据文件。

## 暂不作为主线

- [ ] 不做复杂安全体系：严格 CSP、外部限流、鉴权后台都不是当前重点，除非线上出现实际滥用。
- [ ] 不重构成 React/Vue/Vite，当前静态页更轻，维护成本更低。
- [ ] 不全局删除动画，继续用性能模式分层。
- [ ] 不大改 iOS 壁纸裁切和安全区逻辑，除非实测确认当前方案仍无法铺满。
- [ ] 不做复杂后台管理，收藏数量和维护频率还不值得引入后台。

## 发布前检查

- [ ] 运行 `node scripts\check.js`。
- [ ] 需要让线上用户立即拿到 CSS/JS 改动时，再运行 `node scripts\update-version.js YYYY.MM.DD.N`。
- [ ] 部署后打开 `https://sksir.top/data/app-version.json`，确认版本号。
- [ ] 手机端如果“看起来没变”，先清 Safari/Service Worker 缓存或确认版本是否已更新。
